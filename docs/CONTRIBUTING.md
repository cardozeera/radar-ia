# Guia de Contribuição

Este documento define como contribuir de forma profissional no Radar IA.

## Princípios

- Respeitar a arquitetura e as regras do projeto.
- Manter documentação atualizada.
- Evitar mudanças desnecessárias.
- Priorizar clareza, qualidade e manutenção.

## Padrões de código

- Escrever código limpo e legível.
- Manter separação de responsabilidades.
- Respeitar Services, Repositories e Providers.
- Usar tipagem Pydantic sempre que aplicável.
- Evitar duplicação de lógica.

## Commits

- Usar mensagens claras e objetivas.
- Preferir commits pequenos e bem descritos.
- Agrupar mudanças relacionadas em cada commit.

## Branches

- Criar branches com nomes descritivos.
- Manter o branch principal estável.
- Trabalhar em branches específicas para cada tarefa ou melhoria.

## Code Review

- Revisar impacto sobre arquitetura e documentação.
- Validar compatibilidade com o restante do projeto.
- Manter foco em qualidade e sustentabilidade.
